# ZODL Web Wallet — UX Specification
> На основе анализа 18 скриншотов мобильного приложения ZODL/ZASHI (Zcash ZEC кошелёк)

---

## 1. Дизайн-система

### Цветовая палитра
| Токен              | Значение              | Применение                        |
|--------------------|-----------------------|-----------------------------------|
| `--bg-primary`     | `#1A1A1A`             | Основной фон страниц              |
| `--bg-secondary`   | `#242424`             | Карточки, панели                  |
| `--bg-input`       | `#2E2E2E`             | Поля ввода, кнопки-иконки         |
| `--accent-purple`  | `#5B3DE8`             | Основной акцент (shielded/приват) |
| `--accent-purple-bg` | `#3D2A9E22`         | Подложка shielded-карточек        |
| `--text-primary`   | `#FFFFFF`             | Заголовки, балансы                |
| `--text-secondary` | `#8A8A8A`             | Подписи, плейсхолдеры             |
| `--error`          | `#E84040`             | Ошибки валидации                  |
| `--success`        | `#3DD68C`             | Успешные транзакции               |
| `--warning`        | `#F5A623`             | Предупреждения, syncing           |

### Типографика
- **Font**: Inter или аналог (geometric sans-serif)
- **Balance display**: `font-size: 48–56px; font-weight: 700; letter-spacing: -1px`
- **Page title**: `font-size: 13px; font-weight: 600; letter-spacing: 1.5px; text-transform: uppercase`
- **Body**: `font-size: 14–16px; font-weight: 400`
- **Label**: `font-size: 12px; font-weight: 500; color: var(--text-secondary)`

### Радиусы и отступы
- Карточки: `border-radius: 16px`
- Кнопки: `border-radius: 12px`
- Поля ввода: `border-radius: 10px`
- Base spacing unit: `8px`

### 3.2 Онбординг — Restore: Secret Recovery Phrase

**Маршрут:** `/restore`

```
┌──────────────── RESTORE ────────────────────── [?] ─┐
│                                                      │
│  Secret Recovery Phrase                              │
│  Please type in your 24-word secret recovery         │
│  phrase in the correct order.                        │
│                                                      │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──┐ │
│  │ 1 __ │ │ 2 __ │ │ 3 __ │ │ 4 __ │ │ 5 __ │ │6 │ │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──┘ │
│  ┌──────┐ ┌──────┐ ...  (4 ряда по 6 слов)          │
│  ...                                                 │
│                                                      │
│  Wallet Birthday Height (optional)  [__________]     │
│                                                      │
│              ┌──────────────────────┐                │
│              │        Next          │                │
│              └──────────────────────┘                │
└──────────────────────────────────────────────────────┘
```

**Отличия от мобильной версии (3 колонки → 6 колонок):**
- На широком экране: **6 колонок × 4 ряда** — все 24 слова видны сразу без скролла
- Tab / Enter перемещают фокус к следующему слову
- Автодополнение из словаря BIP-39 (выпадающий список при вводе)
- Кнопка "paste all" — пользователь может вставить всю фразу разом в первое поле, и она автоматически распределяется по ячейкам
- Кнопка [?] открывает `Tooltip-панель` (не нижний шит как в мобайл, а боковой popover)

**Tooltip-контент ("Need to know more?"):**
- Secret Recovery Phrase — 24-слово, BIP-39, мастер-ключ кошелька
- Wallet Birthday Height — номер блока создания кошелька, ускоряет восстановление

**Состояния кнопки Review:**
- `disabled` (серый) — если поля не заполнены или есть ошибки
- `active` (белый) — все поля валидны

**Валидация (inline, real-time):**
- `Invalid address` — красная обводка поля + текст ошибки под полем
- `Insufficient funds` — красная обводка amount + текст ошибки
- При hidden balance режиме ([Eye] off): сумма скрыта дашами `ž — — — —`

**Web-улучшение:**
- Поле Address: автодополнение из Contact Book
- Кнопка [QR Scan] открывает camera API в модале (или кнопка paste)
- Поле Amount: параллельный ввод ZEC / USD (live-конвертация)
- Кнопка [MAX] — автоматически ставит максимально доступную сумму

## 4. Навигация и поток пользователя

```
Onboarding Flow:
/welcome → /create  (новый кошелёк)
         → /restore (восстановление)

Main App Flow:
/dashboard
  ├── /receive          → QR Modal
  ├── /send             → Review Modal → Confirm → Success
  ├── /pay              → Select Asset Modal → Review → Confirm
  ├── /swap             → Get Quote → Review → Confirm
  ├── /wallets          → Connect Hardware
  └── /settings
```

## 6. Модальные окна (Web-специфика)

На мобайле переходы — это отдельные "экраны". На вебе следующее реализуется как **Modal/Dialog**:

| Действие              | Мобайл           | Web                        |
|-----------------------|------------------|----------------------------|
| QR Code view          | Отдельный экран  | Modal overlay              |
| Select Asset          | Bottom sheet     | Centered modal             |
| Need to know more?    | Bottom sheet     | Popover/Tooltip            |
| Hardware Wallet       | Отдельный экран  | Side panel или Modal       |
| Review transaction    | Отдельный экран  | Modal с деталями и confirm |
| Slippage settings     | Inline dropdown  | Popover                    |

## 7. Privacy UX (ключевая особенность Zcash)

- **Shielded адрес** — всегда выделен фиолетовым цветом и иконкой щита
- **Transparent адрес** — нейтральный серый, без акцента
- **Подсказка**: "For privacy, always use shielded address" — закреплённая плашка внизу страницы Receive
- **Hidden balance** — кнопка [Eye] в header доступна глобально
- **Encrypted message** — поле с подписью "encrypted" для поддержания концепции приватности

## 9. Ключевые отличия Web от Mobile

| Аспект               | Mobile                    | Web                                    |
|----------------------|---------------------------|----------------------------------------|
| Навигация            | Нижний tab bar            | Левый sidebar                          |
| Seed phrase grid     | 3 колонки × 8 рядов       | **6 колонок × 4 ряда** (всё на экране) |
| Seed phrase ввод     | Клавиатура по одному слову | Paste all + Tab-навигация              |
| QR Code              | Отдельный экран           | Modal                                  |
| Bottom sheets        | Native gesture sheets     | Modal dialogs                          |
| Balance              | Центрированный            | В Balance Card (левый блок дашборда)  |
| Amount input         | Только ZEC                | ZEC + USD конвертация параллельно      |
| Scan QR              | Камера устройства         | Web Camera API + paste fallback        |
| Hardware wallet      | Bluetooth/USB             | USB (WebUSB API) / Keystone QR         |
